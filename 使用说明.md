# UART 驱动使用说明

## 文件清单

| 文件 | 说明 |
|------|------|
| `Inc/device_err.h` | 公共错误码枚举 |
| `Inc/uart.h` | UART 驱动头文件，结构体与 API 声明 |
| `Src/uart.c` | UART 驱动实现，中断收发逻辑 |
| `Src/uart_demo.c` | 使用例程 |

## 与 CubeMX 的分工

| CubeMX 负责 | 驱动负责 |
|---|---|
| GPIO 时钟 + 引脚配置（PA9/PA10） | IDLE 中断使能 |
| UART 参数（波特率、数据位等） | 接收缓冲管理 |
| NVIC 中断优先级 | HAL 回调（收/发/错） |
| `HAL_UART_Init()` | 应用层 API（send/recv/flush） |

## 快速上手

### 1. CubeMX 配置

1. 启用 USART1，模式设为 **Asynchronous**
2. 参数保持默认（115200, 8N1），因为驱动不自己配置这些
3. **NVIC 设置页 → 勾选 USART1 global interrupt**
4. 生成代码

### 2. 复制驱动文件到工程

CubeMX 生成的项目结构：

```
你的项目/
├── Core/
│   ├── Inc/          ← CubeMX 头文件
│   └── Src/          ← CubeMX 源文件
├── Drivers/          ← HAL 库，不用动
└── 你的项目.uvprojx
```

把驱动文件复制进去：

| 驱动文件 | 放到 |
|---|---|
| `Inc/device_err.h` | `Core/Inc/` |
| `Inc/uart.h` | `Core/Inc/` |
| `Src/uart.c` | `Core/Src/` |

驱动代码本身 **不需要修改**。

### 3. 配置中断入口

在 `stm32f1xx_it.c` 中添加两处代码（写在 `USER CODE` 区域内，CubeMX 重新生成时不会被覆盖）：

**顶部 Includes 区域：**
```c
/* USER CODE BEGIN Includes */
#include "uart.h"
/* USER CODE END Includes */
```

**USART1_IRQHandler 内部：**
```c
void USART1_IRQHandler(void)
{
  /* USER CODE BEGIN USART1_IRQn 0 */
  if (uart_idle_hook(&huart1)) return;
  /* USER CODE END USART1_IRQn 0 */
  HAL_UART_IRQHandler(&huart1);
  ...
```

> 这样驱动的 IDLE 中断处理会在 HAL 之前执行，处理完直接返回，避免 HAL 默认逻辑干扰。

### 4. 在 main.c 中调用

在 `main.c` 顶部添加头文件和变量定义：

```c
/* USER CODE BEGIN Includes */
#include "uart.h"
/* USER CODE END Includes */

/* USER CODE BEGIN PV */
static uint8_t rx_buf[UART_RX_BUF_SIZE];

static uart_base_t uart1 = {
    .huart       = &huart1,             // CubeMX 生成的句柄
    .rx_buf      = rx_buf,
    .rx_buf_size = UART_RX_BUF_SIZE,
};
/* USER CODE END PV */
```

在 `main()` 中初始化和使用：

```c
int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_USART1_UART_Init();    // CubeMX 生成的，必须在 uart_init 之前

    /* USER CODE BEGIN 2 */
    uart_init(&uart1);         // 驱动初始化：使能 IDLE 中断，启动接收链
    /* USER CODE END 2 */

    while (1)
    {
        /* USER CODE BEGIN 3 */
        // 接收示例
        if (uart1.rx_done) {
            uint8_t buf[256];
            uint16_t n = uart_recv(&uart1, buf, sizeof(buf));
            // 处理 buf[0..n-1]
        }

        // 发送示例
        // uint8_t data[] = "hello\r\n";
        // uart_send(&uart1, data, sizeof(data) - 1);

        /* USER CODE END 3 */
    }
}
```

> **注意：** 代码写在 `USER CODE BEGIN/END` 注释之间，这样 CubeMX 重新生成代码时不会被覆盖。

## 接收机制

驱动采用**单字节中断链 + IDLE 中断**实现不定长帧接收：

```
RX 引脚 → RXNE 中断 → 逐字节存入 rx_buf
        → IDLE 中断 → rx_done 置 1，标识一帧完成
        → 应用层通过 uart_recv() 读取整帧
```

- 帧间间隔超过 1 字节时长即触发 IDLE，表示一帧结束
- `rx_done` 置 1 后，应用层应在下次数据到来前及时读出，否则新帧会覆盖旧数据

## CubeMX 重新生成代码后的处理

驱动不依赖 CubeMX 生成的任何文件，重新生成后 **无需手动修改**，只需确保：

| 检查项 | 说明 |
|---|---|
| USART1 的 NVIC 中断 | **要勾选**，由 CubeMX 生成 NVIC 使能代码 |
| `MX_USART1_UART_Init()` | 保留在 `main.c` 中，`uart_init()` 依赖它 |

## API 速查

### 数据结构

```c
typedef struct uart_base {
    UART_HandleTypeDef* huart;       /* CubeMX 生成的句柄 */
    uint8_t*            rx_buf;      /* 接收缓冲（用户分配） */
    uint16_t            rx_buf_size; /* 缓冲容量 */
    volatile uint16_t   rx_len;      /* 已接收字节数 */
    volatile uint8_t    rx_done;     /* 帧完成标志（IDLE 触发） */
} uart_base_t;
```

### 错误码

```c
typedef enum device_err {
    DRV_OK = 0,          /* 成功 */
    DRV_ERR_NULL,        /* 空指针 */
    DRV_ERR_INIT,        /* 初始化失败 */
    DRV_ERR_TIMEOUT,     /* 超时 */
    DRV_ERR_PARAM,       /* 参数错误 */
    DRV_ERR_BUSY,        /* 忙 */
    DRV_ERR_IO,          /* IO 错误 */
} device_err_t;
```

### 函数

| 函数 | 返回值 | 说明 |
|------|--------|------|
| `uart_init(uart)` | `device_err_t` | 使能 IDLE 中断，启动接收。需先调用 `MX_USART1_UART_Init()` |
| `uart_send(uart, data, len)` | `device_err_t` | 阻塞发送，发完才返回 |
| `uart_recv(uart, buf, len)` | `uint16_t` | 读取已接收帧到 buf，返回实际字节数，读后清空 `rx_done` |
| `uart_idle_hook(huart)` | `int` | IDLE 中断预处理，返回 1 表示已处理。**不要直接调用**，由 `stm32f1xx_it.c` 中的 `USART1_IRQHandler` 调用 |

### 使用示例

见 `Src/uart_demo.c`。

关键步骤：`MX_USART1_UART_Init()` → `uart_init()` → 轮询 `rx_done` → `uart_recv()` → `uart_send()`

所有函数返回 `device_err_t`，成功返回 `DRV_OK`，失败返回对应错误码。
